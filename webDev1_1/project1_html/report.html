Project Report

This page documents each site page, the markup and styling used, responsiveness and a short description of functionality and technologies.

Page functionality and technologies
Page Markup Responsive CSS / Frameworks Description
Home HTML5 Yes CSS3 (Flexbox, Grid), Google Fonts, Font Awesome Primary landing page with header, navigation and a weather widget. Uses fetch to call a weather API and displays icons. Responsive layout uses Flexbox and media queries. Hard part: making the header/nav collapse and align consistently across breakpoints.
Stats HTML5 Yes CSS3 Displays mock server statistics. Emphasis on clear data presentation with readable typography and spacing.
News HTML5 Yes CSS3 Aggregates news items and links. Uses anchor elements for external articles and internal navigation to full articles. Emphasis on readable typographic scale and responsive layout.
Notes HTML5 Yes CSS3 Simple notes/clipboard interface for me. Includes form controls and client-side validation. Styling focuses on contrast and touch targets for mobile.
Report (this page) HTML5 Almost CSS3 (table styling) Documentation page describing each site page, technologies used and implementation notes. Accessible from the main menu.

External libraries & tools

  • JS (pls don't blame me)
  • Moment.js for date/time formatting
  • Font Awesome for icons
  • API for external weather requests

Notes on implementation

  • All pages use semantic HTML5 elements for accessibility.
  • Responsive behaviour tested at common breakpoints, media queries handle layout changes.
  • Where JavaScript fetches external data (weather), API keys are loaded from a separate script and not inlined in HTML.
